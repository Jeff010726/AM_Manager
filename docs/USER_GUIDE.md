# 使用文档

## 1. 登录
- 默认管理员：`admin@example.com / admin123`
- 默认访客：`visitor@example.com / visitor123`

## 2. 权限模型
- `admin`：可操作全部功能（主数据、库存、项目、用户）。
- `visitor`：只读；若被加入项目成员，可提交该项目 Commit 更新状态。

## 3. SKU 主数据页
- 仅展示主数据字段：SKU 编号、产品名称、分类、型号/规格、单位、安全库存、状态。
- 不展示库存数量。
- 管理员可新增分类、新增 SKU、删除 SKU。
- 删除 SKU 会同步删除该 SKU 的库存/项目关联数据。

## 4. 库存页
- 仅展示库存口径：总库存、在途、在手、可用、预留、已消耗、安全库存、缺口。
- 点击 SKU 可进入库存详情页：
  - 查看 SKU 信息。
  - 查看库存流水（精确到时间、关联项目、备注、操作人）。
  - 管理员可在流水中执行：
    - 编辑（修改数量/备注）
    - 撤销（自动生成反向流水）
  - 当前仅支持对非项目关联的手工库存流水进行编辑/撤销（入库、出库、创建在途、在途入库、库存调整）。
- 管理员可执行：入库、创建在途、在途入库、项目预留、项目消耗。

## 5. 项目页
- 首屏仅展示项目列表。
- 点击项目名称进入详情页，可查看：
  - 项目成员与角色
  - 项目预留库存明细
  - 项目 Commit 记录（状态流转 + 进度 + 时间）
- 管理员可：新增项目、分配成员、删除项目。
- 项目成员（或管理员）可提交 Commit 更新项目状态。
- 删除项目时，系统会先释放未消耗的预留库存回可用库存。

## 6. 用户页（管理员）
- 可新增用户。
- 可删除用户账号（软删除）：
  - 账号将失效，无法再登录。
  - 历史项目参与和 Commit 仍保留可见。

## 7. 关键业务规则
- 项目消耗前必须先预留。
- 库存写操作使用幂等键，避免重复提交。
- 系统至少保留一个活跃管理员账号。

## 8. 常见错误码
- `401`：Token 无效或过期，请重新登录。
- `403`：权限不足。
- `409`：库存不足、重复幂等键或业务冲突。
- `400`：参数校验失败。
